# Brief: The Context Re-Entry Specialist

**Client:** Me - a solo developer running 3 client projects plus a side build.

**What's broken.** I switch projects constantly: Project A Monday, then Project B
Thursday. Every switch costs 30-45 minutes of cold-start - re-reading old threads,
scanning my own commits, reconstructing where I was and why. That's 2-3 hours a week
of re-orientation tax. Worse, I sometimes restart on the wrong thing because I've
forgotten what was actually blocking me.

**What I've tried.** TODO.txt and sticky notes capture tasks but not state or intent.
End-of-day notes help when I write them, which is rarely. And anything memory-based
has a catch: if I could remember enough to write it down, I've already paid half the
cost - and my memory is often wrong, like thinking I finished something that's
actually uncommitted.

**What I need.** A specialist that reconstructs from the trail I already left - git
history, uncommitted work, branches, stale TODOs - not from memory. It returns a
re-entry map: where I was, the open decision, what's resolved or not, and the single
first action, with the rule it won on. A brain-dump is optional, for intent the
repo can't show. When memory and the repo disagree, it tells me - the repo wins.

**What good looks like.** I point it at the repo, read for 90 seconds, and I'm working
on the right thing - no archaeology, no false-memory traps. A stranger points it at
their repo and gets the same.
